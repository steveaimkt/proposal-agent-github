## 작성 가이드

### 5.1 수행 조직도
- PM → 팀장 → 팀원 계층 구조
- 각 역할의 책임 범위 명시
- 발주처 조직과의 협업 체계 포함
- 예시:
  ```
  [발주처]
      ↕ (협업)
  [PM] - 전체 총괄
    ├─ [기획팀] - 전략/콘텐츠 기획
    ├─ [제작팀] - 콘텐츠 제작
    ├─ [운영팀] - 채널 운영/모니터링
    └─ [분석팀] - 데이터 분석/리포팅
  ```

### 5.2 핵심 인력 프로필
- PM + 핵심 인력 (3-5명)
- 각 인력별:
  - 이름, 역할, 경력 연수
  - 관련 자격증/인증
  - 유사 프로젝트 경험

### 5.3 콘텐츠 검수 & 품질 관리 (★핵심)
- 콘텐츠 검수 프로세스
- 품질 관리 기준
- 승인 워크플로우
- 예시:
  ```
  기획 → 제작 → 1차 검수(내부) → 2차 검수(클라이언트) → 수정 → 최종 승인 → 발행
  ```

### 5.4 퍼포먼스 리포팅 체계 (★핵심)
- 리포트 종류 및 주기
  - 일간: VOC 모니터링
  - 주간: 콘텐츠 성과 리포트
  - 월간: 종합 퍼포먼스 리포트
  - 분기: 전략 검토 리포트
- 리포트 구성 요소:
  - 채널별 KPI 현황
  - 콘텐츠별 성과 분석
  - 인사이트 및 개선 방안
  - 경쟁사 벤치마킹

### 5.5 데이터 분석 & 시각화
- 분석 도구 및 대시보드
- 주요 지표 정의
- 인사이트 도출 프로세스
- 예시:
  - 전체 채널 데이터 요약
  - 채널별 콘텐츠 발행 결과
  - 키 러닝 발굴 → 익월 플랜 반영

### 5.6 커뮤니케이션 체계
- 정기 회의: 주간 회의, 월간 검토 회의
- 보고 채널: 이메일, 협업툴, 대면
- 긴급 연락망
- 예시:
  - 주간 회의: 매주 월요일 10시
  - 월간 리뷰: 매월 첫째 주 수요일
  - 긴급 이슈: 카카오톡 채널 (24시간)

### 5.7 이슈 에스컬레이션
- 이슈 등급 분류 (상/중/하)
- 등급별 대응 시간
- 에스컬레이션 경로

### 5.8 VOC 모니터링 & 위기 대응
- 모니터링 범위 및 키워드
- 위기 등급 분류
- 대응 프로세스
- 대응 시나리오 (FAQ)
