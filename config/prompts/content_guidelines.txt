# 제안서 콘텐츠 작성 가이드라인 (v3.1)

이 문서는 모든 Phase 프롬프트에서 공통으로 참조하는 가이드라인입니다.

## 1. Action Title (인사이트 기반 제목)

### 개념
Topic Title(주제 제목) 대신 Action Title(인사이트 제목)을 사용하여
슬라이드를 읽지 않고도 핵심 메시지를 전달합니다.

### 원칙
- 슬라이드 내용의 결론/인사이트를 제목에 담기
- 숫자나 구체적 데이터 포함 권장
- 동사 또는 명사형 결론문으로 구성
- 15-30자 내외로 간결하게

### 변환 예시
| Topic Title (기존) | Action Title (개선) |
|-------------------|---------------------|
| 사업 개요 및 목표 | 10개월간 브랜드 인지도 +20%p 달성이 핵심 과제 |
| 홍보 환경 분석 | 숏폼·로컬 트렌드가 [프로젝트명] 성장 기회 |
| 타겟 분석 | MZ세대 2030이 핵심, 하루 SNS 55분 사용 |
| SNS 운영 전략 | 인스타그램 집중 운영으로 팔로워 5,000+ 달성 |
| 차별화 포인트 | 유사 사업 130% 목표 초과 달성, 검증된 실행력 보유 |
| 시민참여형 행사 | 연간 31회+ 행사로 시민 5,000명 직접 만남 기회 창출 |

### 피해야 할 표현
- "~~에 대하여"
- "~~의 현황"
- "~~의 방안"
- "~~의 필요성"
- "~~의 개요"

## 2. Win Theme (핵심 수주 전략)

### 개념
Win Theme은 제안서 전체에 반복적으로 등장하여
핵심 차별화 메시지를 각인시키는 전략적 키워드/문장입니다.

### 역할
- 발주처의 Pain Point에 직접 대응
- 경쟁사와의 차별점을 명확히 전달
- 제안서 전체의 일관된 메시지 구조 형성

### 구성 요소
1. **이름** (짧은 키워드): "데이터 기반 타겟 마케팅"
2. **설명** (1문장): "MZ세대 SNS 이용 패턴 분석을 통한 최적화된 콘텐츠 전략"
3. **근거** (Evidence): 구체적 수치, 사례, 역량
4. **적용 Phase**: 어떤 섹션에서 이 Win Theme을 강조할지

### 마케팅/PR 제안서 Win Theme 예시
| Win Theme | 설명 | 주요 적용 Phase |
|-----------|------|----------------|
| 데이터 기반 타겟 마케팅 | MZ세대 SNS 이용 패턴 분석 기반 콘텐츠 최적화 | 1, 3, 6 |
| 시민 참여형 브랜드 빌딩 | 오프라인 커뮤니티로 지속가능 브랜드 구축 | 4 |
| 온-오프라인 통합 시너지 | SNS-행사-홍보물 연계로 노출→참여→확산 선순환 | 2, 5 |
| 검증된 전문성과 실행력 | 유사 사업 수행 실적, KPI 달성률 130% | 7 |

### 적용 방법
1. **섹션 구분자(Divider)**: 하단에 Win Theme 배지로 표시
2. **Executive Summary**: 3대 Win Theme 시각적 강조
3. **각 Phase 슬라이드**: 내용이 Win Theme과 연결되도록 작성

## 3. KPI 산출 근거

### 원칙
모든 KPI는 "어떻게 이 숫자가 나왔는지" 산출 근거를 함께 제시해야 합니다.

### 구조
```
[목표값] = [현재값] + [개선 요소들의 합]

예: 팔로워 +30% = 인플루언서 협업(+10%) + 릴스 확대(+12%) + 이벤트(+8%)
```

### 예시
| KPI | 목표 | 산출 근거 | 출처 |
|-----|------|----------|------|
| 팔로워 성장률 | +30% | 인플루언서 +10%, 릴스 +12%, 이벤트 +8% | 유사 프로젝트 평균 성장률 참고 |
| 연간 도달 | 100만+ | 릴스 도달률 피드 대비 1.8배 | Social Insider 2025 |
| 행사 참여자 | 5,000명+ | 러닝 10회×50명 + 요가 6회×30명 + ... | 자체 운영 실적 기준 |

## 7. 차별화 포인트 작성

### 원칙
"우리가 잘한다"가 아닌 "구체적으로 어떻게 잘하는지"를 증명

### 나쁜 예시
- "인천 지역 전문성" → 막연함

### 좋은 예시
- "인천 특화 네트워크: 인천 인플루언서 47명 확보, 지역 언론사 5개 MOU"

### 구조
| 차별점 | 상세 설명 (구체적 수치) |
|--------|----------------------|
| 인천 특화 네트워크 | 인플루언서 47명, 언론사 5개 MOU, 로컬 크리에이터 협업 3년+ |
| 통합 마케팅 전문성 | SNS 계정 15개+ 운영, 콘텐츠 월 200개+ 제작, 디자이너 4명 내재화 |
| 시민참여 검증된 성과 | 러닝클럽 8,000명+ 운영, 만족도 4.7/5.0, 재참여율 65% |
| 데이터 기반 운영 | 실시간 대시보드, A/B 테스트, KPI 달성률 평균 130% |

## 8. 슬라이드 레이아웃 규칙 (v3.4)

### Zone 시스템 (절대 규칙)
```
타이틀 바:  0.00" ~ 0.88"  (TB 함수가 자동 처리)
콘텐츠:    1.10" ~ 6.50"  (5.4" 가용 — Y0=1.1, YB=6.5)
푸터:      6.70" ~ 7.50"  (PN, source 텍스트)
```

### 간격 규칙 (겹침 방지 — 필수 준수)
| 요소 조합 | 최소 간격 | 이유 |
|-----------|----------|------|
| HIGHLIGHT → IMG_PH | 0.75" | HIGHLIGHT 높이 ~0.65-0.7" |
| HIGHLIGHT → 다음 텍스트 | 0.75" | 동일 |
| METRIC_CARD → 다음 요소 | 0.15" | 카드 하단 여백 |
| COLS → HIGHLIGHT | 0.3" | COLS 하단 여백 |
| MT(불릿) → 다음 요소 | 0.2" | 텍스트 하단 여백 |

### MT(불릿 텍스트) 높이 기준
```
✅ 줄 수에 맞춤:  3줄=1.1"  4줄=1.4"  5줄=1.7"  6줄=2.0"  8줄=2.8"
❌ 고정 높이:     줄 수 무관하게 h=3.2" (하단 공백 발생)
```

### 한글 텍스트 너비 (인치/글자)
| 폰트 크기 | 글자당 너비 | CW(~11.8") 내 최대 글자수 |
|----------|-----------|----------------------|
| 44pt | 0.61" | ~18자 |
| 36pt | 0.50" | ~23자 |
| 28pt | 0.39" | ~30자 |
| 18pt | 0.25" | ~47자 |

**규칙**: 44pt 제목이 18자 초과 시 2줄로 분리 (별도 T() 호출)

### Phase 3 필수 컨셉 장표 (3종)
1. **Concept Reveal** — 다크 배경, 60pt 대형 컨셉 키워드, 4단계 순환 카드
2. **Strategy Synergy Map** — 3대 Win Theme 연결 구조, 순환 흐름도
3. **Big Idea Reveal** — 36pt 중앙 컨셉 + 3-Step 카드

### 공백 보완 규칙
- 콘텐츠 하단 0.5"+ 공백 → IMG_PH 또는 HIGHLIGHT 추가
- METRIC_CARD 높이 확대 (비율 기반 배치가 자동 대응)
- **수정 불필요**: 섹션 구분자, 표지, 마지막, Concept Reveal의 여백은 의도적

### 배경색 충돌 방지
- slide_next_step: 배경 C["dark"] — 카드 색상과 겹치지 않도록
- slide_cover/closing: 배경 C["primary"]
- slide_section_divider: 배경 C["dark"]
- 카드 색상 = 배경 색상인 경우 반드시 다른 색상으로 변경

## 9. 설득 로직 프레임워크: C-E-I (Claim → Evidence → Impact)

### 개념
모든 콘텐츠 슬라이드는 C-E-I 3단 설득 구조를 따릅니다.
평가위원이 "근거가 있는 주장"이라고 느끼게 만드는 것이 목표입니다.

### 구조
```
Claim (주장)    → Action Title에 반영
Evidence (근거) → 데이터, 실적, 사례로 뒷받침
Impact (영향)   → 발주처 관점의 가치/효과
```

### 예시
| 요소 | 나쁜 예 | 좋은 예 |
|------|--------|--------|
| Claim | "효과적으로 운영하겠습니다" | "인스타그램 릴스 중심 전략으로 도달률 3배 확보" |
| Evidence | "경험이 많습니다" | "Social Insider 2025: 릴스 도달률 피드 대비 1.8배, 유사 프로젝트 A사 도달률 280% 향상" |
| Impact | "좋은 결과를 기대합니다" | "월 150만 도달 → 브랜드 인지도 +20%p → 연간 미디어 가치 6.5억원" |

### 적용 규칙
- Claim은 Action Title에 담는다 (슬라이드를 안 읽어도 핵심이 전달됨)
- Evidence는 반드시 1개 이상의 숫자/데이터/출처를 포함한다
- Impact는 발주처의 Pain Point 해결과 연결한다
- Phase 0(HOOK)은 Claim만, Phase 6(WHY US)은 Evidence 중심, Phase 7(ROI)은 Impact 중심

## 10. Phase 간 내러티브 연결 (Bridge)

### 개념
각 Phase가 독립적으로 읽히지 않고, 하나의 스토리로 연결되어야 합니다.
각 Phase의 마지막 슬라이드(또는 다음 Phase의 섹션 구분자)에 브릿지 문장을 포함합니다.

### 브릿지 패턴
| 전환 | 브릿지 패턴 | 예시 |
|------|-----------|------|
| Phase 1→2 | "이 목표를 달성하기 위해 먼저 시장을 이해해야 합니다" | "팔로워 5만 달성을 위해, 현재 시장이 어떻게 변하고 있는지 살펴보겠습니다" |
| Phase 2→3 | "이러한 시장 기회를 잡기 위한 우리만의 전략은..." | "숏폼 트렌드와 로컬 수요, 이 두 기회를 동시에 잡는 전략을 제시합니다" |
| Phase 3→4 | "이 전략을 구체적으로 실행하는 방법은..." | "3C 전략을 채널별로 어떻게 실행하는지 상세히 보여드리겠습니다" |
| Phase 4→5 | "이 실행 계획의 안정적 운영을 보장하는..." | "40개+ 월간 콘텐츠의 품질과 일정을 보장하는 운영 체계입니다" |
| Phase 5→6 | "이 모든 것을 실현할 수 있는 이유는..." | "이 전략과 운영 체계를 검증한 실적을 보여드리겠습니다" |
| Phase 6→7 | "검증된 역량으로 달성할 기대효과는..." | "이 실행력이 만들어낼 구체적인 성과와 ROI입니다" |

### 적용 방법
- 각 Phase의 마지막 슬라이드에 `key_message` 필드로 브릿지 문장 포함
- 또는 섹션 구분자의 `story` 필드에 브릿지 반영

## 11. 평가 기준 정렬 원칙

### 개념
RFP 평가 기준의 배점을 분석하여, 고배점 항목에 제안서 분량과 깊이를 집중합니다.
평가위원이 점수를 매길 때 "이 항목은 확실히 잘했다"고 느끼게 해야 합니다.

### 배점별 대응 전략
| 배점 비중 | 제안서 대응 | 슬라이드 배정 |
|----------|-----------|------------|
| 25% 이상 | 최고 우선순위 — 해당 영역 전용 슬라이드 3장+ | Phase 내 최다 분량 |
| 15~24% | 높은 우선순위 — 전용 슬라이드 2장+ | 충분한 분량 |
| 15% 미만 | 기본 대응 — 1장으로 커버 가능 | 기본 분량 |

### 적용 방법
- RFP 분석 단계에서 `evaluation_strategy`를 추출
- 각 Phase 생성 시 고배점 항목과 매핑하여 강조 지시
- 예: "기술 역량" 배점 30% → Phase 3에서 기술 프레임워크 상세 설명, Phase 6에서 기술 인력 강조

## 12. Win Theme 강화 규칙

### 개념
Win Theme은 제안서 전체에 일관되게 반복되어 핵심 메시지를 각인시킵니다.
"이 회사는 OO가 강하다"는 인상을 평가위원 뇌에 남기는 것이 목표입니다.

### Win Theme 정의 (Phase 1에서 확정)
- 반드시 3개 정의
- 각 Win Theme은 RFP Pain Point에 직접 대응
- 서로 다른 축으로 분산 (데이터/전문성/통합 등)

### Phase별 Win Theme 출현 규칙
| Phase | Win Theme 역할 | 출현 방식 |
|-------|--------------|----------|
| 0 (HOOK) | 암시 | 3가지 핵심 약속을 암시하는 메시지 |
| 1 (SUMMARY) | **정의** | Win Theme 3개 명확 정의 + Executive Summary |
| 2 (INSIGHT) | 문제 연결 | 각 인사이트가 Win Theme의 "왜 필요한가"와 연결 |
| 3 (CONCEPT) | 전략 연결 | Win Theme을 실현하는 전략 프레임워크 제시 |
| 4 (ACTION) | 실행 연결 | 각 실행 항목이 어떤 Win Theme을 구현하는지 명시 |
| 5 (MANAGEMENT) | 보장 | Win Theme 달성을 보장하는 관리 체계 |
| 6 (WHY US) | **증명** | Win Theme별 증거(실적/역량) 매핑 |
| 7 (INVESTMENT) | 효과 연결 | Win Theme별 ROI/기대효과 |

### Win Theme Badge
각 섹션 구분자에 관련 Win Theme을 표시합니다:
```
win_theme_key="data"  → "데이터 기반 타겟 마케팅" 뱃지 표시
```
