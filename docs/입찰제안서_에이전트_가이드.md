```

│  STEP 1  RFP 문서 파싱                                       │
│  PDF/DOCX → 텍스트, 테이블, 섹션 자동 추출                   │
│  (pypdf + pdfplumber)                                        │
└──────────────────────┬───────────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────────┐
│  STEP 2  RFP 전략 분석 (Claude AI)                           │
│  ├─ 프로젝트 유형 자동 분류 (6종)                            │
│  ├─ Pain Point 추출 (RFP 행간 읽기)                          │
│  ├─ Win Theme 후보 3개 도출                                  │
│  └─ 평가 기준 → 제안서 강조 포인트 변환                      │
└──────────────────────┬───────────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────────┐
│  STEP 3  콘텐츠 생성 (Claude AI × 8 Phase)                   │
│  Phase 0 HOOK → Phase 1 SUMMARY → ... → Phase 7 INVESTMENT  │
│  ├─ Win Theme 전달 체인 (Phase 1에서 확정 → 2~7에 전달)      │
│  ├─ Action Title (인사이트 기반 제목)                         │
│  ├─ C-E-I 설득 구조 (주장→근거→영향)                         │
│  └─ KPI + 산출 근거                                          │
└──────────────────────┬───────────────────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────────────────┐
│  STEP 4  PPTX 렌더링 (slide_kit.py 엔진)                    │
│  ├─ Modern 디자인 시스템                                     │
│  ├─ 20가지 레이아웃 프리셋 자동 선택                         │
│  ├─ 차트(BAR/PIE/LINE), 다이어그램, 간트 차트                │
│  └─ VStack 자동 배치 (겹침 방지)                             │
│                                                              │
│  → 40~80장 완성된 PPTX 출력                                  │
└──────────────────────────────────────────────────────────────┘
```

## 3. Impact-8 Framework — 제안서 8-Phase 구조

실제 수주 성공 제안서를 분석하여 도출한 **검증된 구조**

```
┌─────────────────────────────────────────────────────────────┐
│  Phase 0: HOOK (티저)                         3~10p (5%)    │
│  → 임팩트 있는 오프닝, "왜 지금인가?" 선언                   │
├─────────────────────────────────────────────────────────────┤
│  Phase 1: EXECUTIVE SUMMARY                   3~5p (5%)     │
│  → 의사결정자용 5분 요약 + Win Theme 3개 정의                │
├─────────────────────────────────────────────────────────────┤
│  Phase 2: INSIGHT                             8~15p (12%)   │
│  → 시장 환경 + Pain Point + As-Is vs To-Be                  │
├─────────────────────────────────────────────────────────────┤
│  Phase 3: CONCEPT & STRATEGY                  8~15p (12%)   │
│  → 핵심 컨셉 + 차별화 전략 + 경쟁 우위                      │
├─────────────────────────────────────────────────────────────┤
│  Phase 4: ACTION PLAN  ★핵심★                 30~60p (40%)  │
│  → 상세 실행 계획 + 콘텐츠 예시 + 채널별 전략               │
├─────────────────────────────────────────────────────────────┤
│  Phase 5: MANAGEMENT                          6~12p (8%)    │
│  → 전담 조직 + 품질 관리 + 리포팅 체계                       │
├─────────────────────────────────────────────────────────────┤
│  Phase 6: WHY US                              8~15p (12%)   │
│  → 수행 역량 + 유사 실적 + 케이스 스터디                     │
├─────────────────────────────────────────────────────────────┤
│  Phase 7: INVESTMENT & ROI                    4~8p (6%)     │
│  → 투자 비용 + 정량적 기대효과 + ROI                         │
└─────────────────────────────────────────────────────────────┘
```

**핵심 포인트**: Phase 4 (ACTION PLAN)이 전체의 **40%** — 실행 계획이 제안서의 핵심

## 4. 에이전트 구축 방식

### 4-1. 시스템 아키텍처

```
main.py (CLI)
  └─ ProposalOrchestrator (워크플로우 조율)
       ├─ PDFParser (문서 파싱)
       ├─ RFPAnalyzer (RFP 분석 에이전트)
       │    └─ Claude Sonnet 4 API
       └─ ContentGenerator (콘텐츠 생성 에이전트)
            └─ Claude Sonnet 4 API (Phase별 8회 호출)
                 └─ ProposalContent (Pydantic 스키마)
                      └─ PPTXGenerator + slide_kit.py
                           └─ PPTX 파일 출력
```

### 4-2. 핵심 컴포넌트 5개

| # | 컴포넌트 | 파일 | 역할 |
|---|---------|------|------|
| 1 | **PDF 파서** | `src/parsers/pdf_parser.py` | pypdf + pdfplumber로 RFP 텍스트/테이블 추출 |
| 2 | **RFP 분석 에이전트** | `src/agents/rfp_analyzer.py` | Claude로 Pain Point, Win Theme, 평가 전략 도출 |
| 3 | **콘텐츠 생성 에이전트** | `src/agents/content_generator.py` | Claude로 8-Phase 슬라이드 콘텐츠 생성 |
| 4 | **데이터 스키마** | `src/schemas/proposal_schema.py` | Pydantic 모델 — 에이전트↔렌더링 인터페이스 |
| 5 | **PPTX 렌더링 엔진** | `src/generators/slide_kit.py` | 2,270줄 — 도형, 차트, 레이아웃, 테마 |

### 4-3. AI 에이전트 설계 원칙

#### RFP 분석 에이전트 — "행간을 읽는다"

```
RFP에 적힌 것                     에이전트가 추출하는 것
─────────────────                 ─────────────────────
"~해야 한다"          →           Pain Point (고민)
"~이 필요하다"        →           숨겨진 니즈
평가 기준 배점         →           제안서 강조 포인트
과업 범위             →           Win Theme 후보 3개
```

#### 콘텐츠 생성 에이전트 — "설득 구조를 심는다"

**Win Theme 전달 체인** (v3.6 핵심 혁신):
```
Phase 1에서 Win Theme 3개 확정
    ↓
Phase 2~7 생성 시 Win Theme을 컨텍스트로 전달
    ↓
모든 Phase가 동일한 핵심 메시지를 반복·강화
    ↓
평가위원이 일관된 설득 논리를 경험
```

**C-E-I 설득 프레임워크**:
```
C (Claim)    → Action Title에 핵심 주장
E (Evidence) → 데이터, 실적, 사례로 뒷받침
I (Impact)   → 발주처에 미치는 가치/효과

예시:
  C: "숏폼 60초 영상이 관광 의사결정 73% 좌우"
  E: Google Travel Insights 2025 데이터
  I: 영상 중심 전략 전환 → 관광객 유치 극대화
```

### 4-4. 프롬프트 엔지니어링 구조

```
config/prompts/
├── content_guidelines.txt   ← 전체 공통 규칙 (Action Title, Win Theme, KPI)
├── phase0_hook.txt          ← Phase별 전문 프롬프트
├── phase1_summary.txt
├── phase2_insight.txt
├── phase3_concept.txt
├── phase4_action.txt        ← 가장 상세 (ACTION이 40% 비중)
├── phase5_management.txt
├── phase6_whyus.txt
└── phase7_investment.txt
```

**프롬프트 구성 3단계**:
1. **시스템 프롬프트** — Phase별 전문 지침 (역할, 원칙, 슬라이드 구성, 체크리스트)
2. **사용자 메시지** — RFP 분석 결과 + Pain Point + Win Theme + 평가 전략 + JSON 형식 지정
3. **공통 규칙 주입** — Action Title 강제 + C-E-I 구조 + KPI 산출 근거

## 5. slide_kit.py — PPTX 렌더링 엔진 (2,270줄)

### 5-1. 디자인 시스템

**컬러 팔레트** (커스텀 디자인 시스템):
```
Primary    #002C5F  다크 블루     ← 주 색상
Secondary  #00AAD2  스카이 블루   ← 보조 색상
Teal       #00A19C  틸           ← Win Theme 뱃지
Accent     #E63312  레드         ← 강조
```

**타이포그래피**: Pretendard (Light ~ Bold)

**레이아웃**: 16:9 (1920 × 1080)

### 5-2. 주요 함수 카테고리

| 카테고리 | 대표 함수 | 용도 |
|---------|-----------|------|
| **슬라이드 템플릿** | `slide_cover()`, `slide_toc()`, `slide_section_divider()` | 표지, 목차, 섹션 구분 |
| **도식화** | `COLS()`, `FLOW()`, `PYRAMID()`, `MATRIX()`, `COMPARE()` | 컬럼, 프로세스, 피라미드 |
| **데이터** | `TABLE()`, `KPIS()`, `STAT_ROW()`, `METRIC_CARD()` | 표, KPI, 통계 |
| **차트** | `BAR_CHART()`, `PIE_CHART()`, `LINE_CHART()` | 바, 파이, 라인 차트 |
| **일정** | `GANTT_CHART()`, `TIMELINE()` | 간트, 타임라인 |
| **조직** | `ORG()` | 조직도 |
| **텍스트** | `T()`, `MT()`, `HIGHLIGHT()`, `QUOTE()` | 텍스트, 불릿, 강조 |
| **이미지** | `IMG_PH()`, `IMG()` | 이미지 플레이스홀더 |
| **자동 배치** | `VStack` | Y좌표 자동 계산, 겹침 방지 |

### 5-3. 레이아웃 선택 가이드

| 콘텐츠 유형 | 사용 함수 | 예시 |
|------------|----------|------|
| 시장 분석 | `STAT_ROW()` + `COLS()` | 핵심 수치 → 분석 카드 |
| 전략 프레임워크 | `PYRAMID()` or `COLS()` | 3단계 전략 구조 |
| 실행 로드맵 | `GANTT_CHART()` | 월별 실행 일정 |
| 예산/비중 | `TABLE()` + `BAR_CHART()` | 표 + 막대 그래프 |
| KPI/목표 | `KPIS()` | 대형 숫자 카드 |
| 비교 분석 | `COMPARE()` or `TABLE()` | AS-IS vs TO-BE |
| 프로세스 | `FLOW()` | 4~5단계 흐름도 |
| 팀 구성 | `ORG()` | 조직도 |

## 6. 프로젝트 유형별 자동 적응

| 유형 | Phase 4 비중 | 특화 콘텐츠 |
|------|-------------|-------------|
| **마케팅/PR** | 40% | 채널별 전략, 콘텐츠 예시, 인플루언서, 릴스/숏폼 |
| **이벤트** | 45% | 공간 설계, 프로그램표, 참가자 여정 |
| **IT/시스템** | 35% | 시스템 아키텍처, WBS, 간트, 마이그레이션 |
| **공공** | 30% | RFP 대응표, 정책 연계, 유사사업 증빙 |
| **컨설팅** | 30% | 전략 프레임워크, 벤치마킹, Quick Win |

에이전트가 RFP를 읽고 **프로젝트 유형을 자동 분류** → Phase별 비중과 특화 콘텐츠 자동 조정

## 7. 사용 가이드

### 7-1. 사전 준비

```bash
# 1. 의존성 설치
pip install -r requirements.txt

# 2. .env 설정
echo "ANTHROPIC_API_KEY=sk-ant-..." > .env
```

### 7-2. CLI 실행

```bash
# 제안서 생성 (기본)
python main.py generate input/rfp.pdf -n "프로젝트명" -c "발주처명"

# 유형 지정
python main.py generate input/rfp.pdf -n "프로젝트명" -c "발주처" -t marketing_pr

# RFP 분석만
python main.py analyze input/rfp.pdf
```

### 7-3. Claude Code 연동 (현재 운영 방식)

실제 운영에서는 **Claude Code가 직접 스크립트를 작성**하는 방식:

```
사용자: "제안요청서/테스트 02 폴더 내 파일을 분석한 후 제안서를 제작해줘"

Claude Code가 수행하는 작업:
  1. RFP PDF를 직접 읽고 분석
  2. Impact-8 구조로 콘텐츠 기획
  3. output/테스트 02/generate_제안서.py 스크립트 생성
  4. slide_kit.py를 import하여 PPTX 생성 코드 작성
  5. 스크립트 실행 → PPTX 출력
  6. QC (도형 넘침, 공백, 차트 부족) → 수정 → 재생성
```

### 7-4. 폴더 구조

```
제안요청서/테스트 XX/        ← RFP 입력 (PDF 파일)
output/테스트 XX/            ← 출력
  ├── generate_제안서.py     ← PPTX 생성 스크립트
  └── 프로젝트명_제안서.pptx ← 최종 결과물
```

## 9. 핵심 혁신 요소 정리

### 9-1. 전략 레이어 (v3.6)

| 혁신 | 설명 | 효과 |
|------|------|------|
| **Pain Point 추출** | RFP "행간"에서 발주처의 진짜 고민 도출 | "우리를 이해한다" 신뢰 구축 |
| **Win Theme 전달 체인** | Phase 1에서 확정 → 2~7에 일관 반복 | 평가위원에게 핵심 메시지 각인 |

| **Action Title** | 모든 제목이 인사이트/결론 기반 | 제목만 읽어도 핵심 전달 |
| **C-E-I 설득 구조** | 주장→근거→영향 3단계 | 논리적 설득력 극대화 |
| **KPI 산출 근거** | 모든 목표에 "왜 이 숫자인지" 포함 | 허황된 약속 방지 |

| 혁신 | 설명 |
|------|------|
| **VStack 자동 배치** | Y좌표 자동 계산으로 도형 겹침 완전 방지 |

| **네이티브 차트** | BAR/PIE/LINE 차트를 PPTX 네이티브로 생성 |
| **커스텀 디자인 시스템** | 컬러, 폰트, 레이아웃 일관성 자동 유지 |

## 10. 기술 스택

