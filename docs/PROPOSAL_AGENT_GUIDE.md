# 입찰 제안서 자동 생성 에이전트 가이드 (v3.1)

## 1. 시스템 개요

### 1.1 목적
RFP(제안요청서) 문서를 입력받아 전문 컨설팅 수준의 PPTX 입찰 제안서를 자동 생성하는 Python 에이전트 시스템

### 1.2 핵심 가치
- **설득력 있는 구조**: Impact-8 Framework 기반 8단계 논리 전개
- **전략적 메시지**: Win Theme을 통한 일관된 수주 전략
- **데이터 기반**: KPI 산출 근거를 통한 신뢰성 확보
- **프로페셔널 디자인**: Modern 디자인 시스템 적용

### 1.3 역할 분리

| 역할 | 담당 | 주요 기능 |
|------|------|----------|
| **Claude Code** | 콘텐츠 생성 | RFP 분석, Phase 0~7 콘텐츠 생성, 전략 도출 |
| **[회사명]** | 문서화 | PPTX 변환, 디자인 적용, 차트/다이어그램 생성 |

## 3. Impact-8 Framework

### 3.1 8단계 구조

실제 수주 제안서 분석을 기반으로 설계된 설득력 있는 8단계 구조

```
┌─────────────────────────────────────────────────────────────────────────┐
│  PHASE 0: HOOK (티저)                                    3-10p (5%)    │
│  ─────────────────────────────────────────────────────────────────────  │
│  목적: 첫 인상에서 승부                                                  │
│  내용: 임팩트 있는 오프닝, 핵심 메시지, 비전 제시                          │
│  핵심: "이 제안서를 끝까지 보고 싶다"는 생각을 유발                        │
├─────────────────────────────────────────────────────────────────────────┤
│  PHASE 1: SUMMARY (Executive Summary)                    3-5p (5%)     │
│  ─────────────────────────────────────────────────────────────────────  │
│  목적: 의사결정권자를 위한 5분 요약                                       │
│  내용: 프로젝트 목표, Win Theme, 핵심 KPI, Why Us                        │
│  핵심: 바쁜 의사결정권자가 핵심을 즉시 파악                               │
├─────────────────────────────────────────────────────────────────────────┤
│  PHASE 2: INSIGHT (시장 환경 & 문제 정의)                8-15p (10%)    │
│  ─────────────────────────────────────────────────────────────────────  │
│  목적: "고객의 상황을 정확히 이해하고 있다" 입증                          │
│  내용: 시장 트렌드, 문제 정의, 숨겨진 니즈, 기회 발굴                     │
│  핵심: 트렌드 → 문제 → 기회 연결 스토리                                  │
├─────────────────────────────────────────────────────────────────────────┤
│  PHASE 3: CONCEPT & STRATEGY (핵심 컨셉)                 8-15p (12%)    │
│  ─────────────────────────────────────────────────────────────────────  │
│  목적: 차별화된 해결 방향 제시                                           │
│  내용: 핵심 컨셉, 전략 방향, 경쟁 우위, 솔루션 개요                       │
│  핵심: "이런 접근은 처음이다"라는 신선함                                  │
├─────────────────────────────────────────────────────────────────────────┤
│  PHASE 4: ACTION PLAN (★핵심)                           30-60p (40%)   │
│  ─────────────────────────────────────────────────────────────────────  │
│  목적: 실행 가능성과 전문성 입증                                         │
│  내용: 상세 실행 계획, 콘텐츠 예시, 채널별 전략, 일정                     │
│  핵심: 구체적 예시로 "실제로 할 수 있다" 증명                             │
├─────────────────────────────────────────────────────────────────────────┤
│  PHASE 5: MANAGEMENT (운영 & 관리)                       6-12p (10%)    │
│  ─────────────────────────────────────────────────────────────────────  │
│  목적: 안정적 운영 체계 제시                                             │
│  내용: 조직 구성, 운영 프로세스, 품질관리, 리포팅 체계                    │
│  핵심: "믿고 맡길 수 있다"는 신뢰감                                      │
├─────────────────────────────────────────────────────────────────────────┤
│  PHASE 6: WHY US (수행 역량)                             8-15p (12%)    │
│  ─────────────────────────────────────────────────────────────────────  │
│  목적: "왜 우리인가"에 대한 확실한 답변                                   │
│  내용: 유사 실적, 전문 인력, 레퍼런스, 차별화 포인트                      │
│  핵심: 검증된 실적으로 리스크 해소                                       │
├─────────────────────────────────────────────────────────────────────────┤
│  PHASE 7: INVESTMENT & ROI (투자 & 효과)                 4-8p (6%)      │
│  ─────────────────────────────────────────────────────────────────────  │
│  목적: 투자 가치 설득                                                   │
│  내용: 비용 구조, 정량적 효과, ROI 분석, Next Step                       │
```

## 4. v3.1 핵심 컴포넌트

### 4.1 Win Theme (수주 전략 메시지)

**정의**: 제안서 전체에 반복되는 3대 핵심 수주 전략 메시지

**목적**:
- 핵심 차별화 메시지를 평가위원 뇌리에 각인
- 일관된 전략적 스토리텔링
- 경쟁사 대비 명확한 포지셔닝

**구조**:
```python
WIN_THEMES = {
    "data": "데이터 기반 타겟 마케팅",
    "community": "시민 참여형 브랜드 빌딩",
    "integration": "온-오프라인 통합 시너지",
}
```

**적용 방법**:
1. Executive Summary에서 3대 Win Theme 정의
2. 각 Phase 섹션 구분자에 관련 Win Theme 표시
3. 슬라이드 내 Win Theme 뱃지로 연결 강조
4. Closing에서 Win Theme 재강조

**예시 (프로젝트 유형별)**:

| 유형 | Win Theme 1 | Win Theme 2 | Win Theme 3 |
|------|------------|------------|------------|
| 마케팅/PR | 데이터 기반 타겟 마케팅 | 시민 참여형 브랜드 빌딩 | 온-오프라인 통합 시너지 |
| 이벤트 | 몰입형 경험 설계 | 안전하고 체계적인 운영 | 지속가능한 레거시 |
| IT/시스템 | 검증된 기술력 | 안정적 운영 체계 | 확장 가능한 아키텍처 |

### 4.2 Action Title (인사이트 기반 제목)

**정의**: Topic Title(주제 제목) 대신 인사이트/결론을 담은 제목

**목적**:
- 슬라이드를 읽지 않아도 핵심 메시지 전달
- 평가위원의 빠른 내용 파악 지원
- 설득력 있는 논리 전개

**변환 원칙**:
1. 슬라이드 내용의 결론/인사이트를 제목에 담기
2. 숫자나 구체적 데이터 포함 권장
3. 동사 또는 명사형 결론문으로 구성
4. 15-30자 내외로 간결하게

**Before/After 예시**:

| Before (Topic Title) | After (Action Title) |
|---------------------|---------------------|
| 사업 개요 및 목표 | 10개월간 브랜드 인지도 +20%p 달성이 핵심 과제 |
| 홍보 환경 분석 | 숏폼·로컬 트렌드가 성장 기회 |
| 타겟 분석 | MZ세대 2030이 핵심, 하루 SNS 55분 사용 |
| SNS 운영 전략 | 인스타그램 집중 운영으로 팔로워 5,000+ 달성 |
| 예산 계획 | 월 3,000만원으로 팔로워 50만 달성 |
| 차별화 포인트 | 유사 사업 130% 목표 초과, 검증된 실행력 보유 |

**피해야 할 표현**:
- ❌ "~~에 대하여"
- ❌ "~~의 현황"
- ❌ "~~의 방안"
- ❌ "~~의 필요성"

### 4.4 KPI 산출 근거

**정의**: 모든 KPI에 "어떻게 이 목표를 도출했는지" 명시

**목적**:
- 목표의 현실성 입증
- 평가위원의 신뢰 확보
- 달성 전략의 구체성 제시

**구조**:
```
목표: [정량적 목표]
산출 근거: [세부 전략별 기여도 합산]
데이터 출처: [참고한 벤치마크/리서치]
```

**예시**:
```
목표: 팔로워 +30% (3,500명 → 4,550명)
산출 근거:
  • 인플루언서 협업: +10% (월 2회 × 10개월)
  • 릴스 콘텐츠 확대: +12% (주 3회 → 주 5회)
  • 참여형 이벤트: +8% (분기별 대형 이벤트)
데이터 출처: 유사 프로젝트 평균 성장률 참고 ([공공기관] 2024)
```

```python
class ProposalContent(BaseModel):
```

## 8. 품질 체크리스트

### 8.1 콘텐츠 체크리스트

- [ ] 모든 슬라이드 제목이 Action Title인가?
- [ ] Win Theme이 3개 정의되어 있는가?
- [ ] Win Theme이 각 Phase에 일관되게 반영되어 있는가?
- [ ] 모든 KPI에 산출 근거가 있는가?
- [ ] Executive Summary가 5분 내 핵심 파악 가능한가?
- [ ] Next Step에 구체적인 일정이 있는가?
- [ ] Placeholder가 `[대괄호]` 형식으로 통일되어 있는가?

### 8.2 설득력 체크리스트

- [ ] "왜 이 프로젝트가 중요한가?"가 명확한가? (HOOK)
- [ ] "무엇을 달성할 것인가?"가 숫자로 표현되어 있는가? (SUMMARY)
- [ ] "시장을 이해하고 있는가?"가 입증되어 있는가? (INSIGHT)
- [ ] "차별화된 접근인가?"가 신선하게 느껴지는가? (CONCEPT)
- [ ] "실제로 할 수 있는가?"가 구체적으로 증명되어 있는가? (ACTION)
- [ ] "믿고 맡길 수 있는가?"가 체계적으로 보이는가? (MANAGEMENT)
- [ ] "왜 우리인가?"가 경쟁사 대비 명확한가? (WHY US)
- [ ] "투자 가치가 있는가?"가 ROI로 설득되는가? (INVESTMENT)

### 8.3 디자인 체크리스트

- [ ] 컬러가 Modern 팔레트를 따르는가?
- [ ] 폰트 크기가 일관성 있는가?
- [ ] 여백이 충분한가?
- [ ] 섹션 구분자가 시각적으로 명확한가?
- [ ] Win Theme 뱃지가 적절히 배치되어 있는가?
